# Facilitator checklist

Use this path for a research pilot or demo. In-app: **Guide** (`/guide`).

## Before the session

1. Set language (EN / PT / FI) in the sidebar  
2. Choose **Live** (or **Demo** only to pick sample questions — answers are still live AI)  
3. Confirm Agents + Models if you changed assignments  

## Run

| Step | Where | What to do |
|------|--------|------------|
| 1 | **Workspace** | Enter one research question → Ask agents (parallel recommended for compare) |
| 2 | **Report** | Open the latest unique question → skim theory answers |
| 3 | **Compare** | Add human guest answers → save |
| 4 | **Compare** | Score pre/post solutions on the rubric (two coders if possible) |
| 5 | **Matrix** | Check honest sources (`answer` vs `schema_default` / `missing`) |
| 6 | **Presentation** | Academic deck: Topic → Intro → Key concepts → Case → Conclusion → Sources |
| 7 | **Export** | Download JSON/CSV and/or **rubric CSV** for analysis |

## Integrity reminders

- Do not treat profile defaults as empirical findings (Matrix shows source tags)  
- HITL notes in sequential mode are injected into the next stage prompt  
- Rubric scores support inter-rater agreement; export before claiming reliability  

## After

- Keep notes on condition (baseline / single / parallel / sequential)  
- Store participant IDs without personal identifiers beyond study protocol
